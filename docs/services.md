# 서비스 구성

이 문서는 `family-archive-appliance`를 운영하는 데 필요한 서비스 전체를 정리합니다.

서비스는 Host OS 수준과 Docker 컨테이너 수준으로 나눕니다.

---

## 1. Host OS 서비스

Docker 컨테이너 밖에서 Host OS가 직접 관리하는 서비스입니다.

| 서비스 | 역할 |
|---|---|
| SSH | 서버 원격 관리, 관리자 SFTP 접근 |
| Samba | 메인 PC와 로컬 네트워크 파일 공유 |
| Docker daemon (systemd 관리) | 컨테이너 실행 환경, 재부팅 및 크래시 시 자동 복구 |
| DDNS 클라이언트 | 외부 도메인과 현재 IP를 동기화 |
| nftables | 호스트 방화벽, 허용 포트 외 차단 |
| Cron | 백업, 정리 등 스케줄 작업 실행 |
| smartmontools | HDD SMART 상태 모니터링 |

Docker daemon은 systemd 서비스로 등록합니다. daemon 크래시 또는 서버 재부팅 시 systemd가 daemon을 재시작하고, `restart: unless-stopped` 설정된 컨테이너가 자동으로 복구됩니다.

nftables는 호스트 수준에서 포트를 제한합니다. 라우터 NAT가 1차 방어선이고 nftables는 2차 방어선입니다. 기본 정책은 SSH 포트, 8443(HTTPS)만 허용합니다.

SFTP는 별도 서비스 없이 SSH 위에서 동작합니다. 관리자가 파일 탐색이나 다운로드에 사용하며, 일반 가족 구성원의 접근 경로가 아닙니다.

Samba는 로컬 네트워크 전용입니다. 포트(445)는 외부에 열지 않으며 nftables에서 로컬 네트워크 대역만 허용합니다.

---

## 2. Docker 컨테이너

| 컨테이너 | 역할 |
|---|---|
| `nginx` | 리버스 프록시, HTTPS 종단, 라우팅 |
| `family-album-api` | REST API 서버, 썸네일 및 리사이즈 이미지 생성 포함 |
| `family-album-web` | React 웹 UI |
| `mongodb` | 메타데이터 저장소 |
| `backup-runner` | 백업 스크립트 실행 |

썸네일과 리사이즈 이미지 생성은 업로드 시 API 내에서 동기로 처리합니다. 별도 워커 컨테이너는 두지 않습니다.

메타데이터 저장소로 MongoDB를 사용합니다. (ADR-002 참고)

---

## 3. 외부 접근 구조

가족 구성원(조부모, 친척 포함)이 외부에서 접근할 수 있어야 합니다.

```
외부 사용자: https://yourdomain.com (443)
  └── 라우터 NAT: 외부 443 → 서버 8443
        └── nginx: 8443에서 수신, HTTPS 종단
              ├── /api  → family-album-api
              └── /     → family-album-web
```

- nginx는 내부 8443 포트에서 수신합니다. 외부 사용자는 표준 443으로 접근하며 포트를 의식하지 않습니다.
- HTTPS는 필수입니다. 인터넷에 열린 서비스에서 HTTP는 사용하지 않습니다.
- 인증서는 Let's Encrypt를 사용합니다. 발급 시 HTTP-01 챌린지(포트 80 필요) 또는 DNS-01 챌린지 중 선택합니다.
- SSH는 22 대신 비표준 포트를 사용합니다. 자동화 스캔 노이즈를 줄이기 위함입니다. 정확한 포트는 서버 설치 시 확정합니다.
- nginx 설정은 직접 관리합니다.

---

## 4. 인증 구조

복잡한 사용자 계정 관리는 초기 범위 밖입니다. 역할별 공용 비밀번호를 API에서 처리합니다.

| 역할 | 권한 | 사용자 |
|---|---|---|
| `uploader` | 업로드, 조회, 다운로드 | 본인, 아내 |
| `viewer` | 조회, 다운로드 | 일가 |

각 역할에 공용 비밀번호를 하나씩 부여하고, 해당 비밀번호를 가족 간에 공유합니다. 개인별 계정 분리는 하지 않습니다.

인증 처리는 API 레이어에서 담당합니다. nginx basic auth는 사용하지 않습니다.

---

## 5. 파일 입력 경로

사진 업로드는 Web UI를 통한 HTTP 업로드만 지원합니다.

SFTP는 관리자가 파일을 직접 탐색하거나 다운로드할 때 사용합니다. 업로드 경로로는 사용하지 않습니다.
